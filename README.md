# Arch-Modernizer
用于自动化重构遗留代码库
## 成果
我构建了一个名为 「Arch-Modernizer」 的多 Agent 协作系统，用于自动化重构遗留代码库。
## 核心解决两大痛点
### 1.人工重构时“不敢改、改不完、改出 Bug 难追溯”的风险；
### 2.大型代码库中散落的技术债（如过时 API、循环依赖）难以系统性清理，规范落地依赖人工 Code Review，效率极低。
## 核心逻辑流
系统采用 Plan-and-Execute 架构，包含三个协作 Agent 和一个长链推理模块：
### 1.洞察与规划 Agent
首先对整个代码库进行静态分析，生成依赖图谱与调用链。此 Agent 使用长链推理：先识别出“被最多其他模块依赖的核心文件”，将其标记为“高影响变更区”；再反向推导“哪些叶子模块可优先安全重构”。最终输出一份最小化风险的拆解计划（例如“先重构 utils 层，再升级 service 层，最后处理 controller”）。
### 2.重构执行 Agent
严格遵循拆解计划，依次执行代码转换。其特点是带上下文的批量改写：每重构一个文件时，会读取该文件的所有 import、export 以及调用它的上游文件，确保语法转换（如 AngularJS 的 $scope 转为 React Hooks）不会破坏外部契约。
### 3.验证与自我纠错 Agent（关键闭环）
每次重构后自动运行单元测试 + 类型检查。如果失败，此 Agent 会执行自我纠错的循环：
（1）捕获错误栈和失败测试用例
（2）回溯到最近一次变更的 diff
（3）提出最多 3 种修复假设（例如“缺少依赖导入”、“变量名未同步”），并逐一尝试，若 3 次重试后仍失败，则回滚该文件变更，并在最终报告中标记为“需人工介入”
### 4.报告生成 Agent
汇总所有成功/失败的重构记录，生成一份带影响面分析的 PR，内容包括：变更文件列表、每个文件的测试通过率、预估的运行时性能变化，以及留给人工审查者的“重点关注建议”。
## 落地成果
目前该系统已在我们后端团队（15 人）和前端团队（8 人）落地，累计处理 12 万行 AngularJS 和 8 万行 Python 2.7 代码。
## 核心数据
 1. 自动化完成 87% 的重构任务，剩下 13% 因涉及业务强耦合逻辑被安全回滚并转人工
 2. 人工 Code Review 时长降低 70%（从平均 4 小时降至 1.2 小时）
 3. 重构后 线上 Bug 率下降 55%（主要归因于自动补全了之前缺失的单元测试）
 4. 每日运维消耗约 180 万 Token，折合成本约 2.3 美元/天，远低于人工重构的人力成本
